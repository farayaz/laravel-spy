# Contributing

Contributions are welcome.

## Local Development

1. Fork and clone the repository.
2. Create a feature branch.
3. Install dependencies.
4. Run tests.
5. Run code style checks.

## Useful Commands

```bash
vendor/bin/phpunit
vendor/bin/pint
```

See `README_TESTING.md` for more details about testing workflows.

## Pull Request Checklist

- Focused and clearly scoped changes.
- Tests added or updated when behavior changes.
- Documentation updated when adding/changing features.
